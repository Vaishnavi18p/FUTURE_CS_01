# Web Application Security Assessment

## Project Overview
This project presents a vulnerability assessment conducted on a publicly accessible web application as part of my cybersecurity internship.

## Target Website
demo.testfire.net

## Objective
To identify common web security issues using passive analysis techniques and provide recommendations.

## Tools Used
- Nmap
- OWASP ZAP (Passive Scan)
- Browser Developer Tools

## Scope
- Public-facing pages only
- No exploitation performed
- Passive testing approach

## Key Findings
- Website not using HTTPS (High)
- Missing Security Headers (Medium)
- Absence of Anti-CSRF Tokens (Medium)
- Cookie Security Issues (Low)
- Open Ports Detected (Low)
- Server Information Disclosure (Low)

## Files Included
- Vulnerability Assessment Report (PDF)
- Evidence screenshots

## ⚠️ Disclaimer
This assessment was conducted ethically using passive techniques for educational purposes only.
